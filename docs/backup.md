# 备份与恢复

OpenPass 提供多层次的备份机制，确保您的密钥数据安全。

---

## 备份策略概览

| 备份类型 | 触发时机 | 存储位置 | 加密 | 用途 |
|---------|---------|---------|------|------|
| 本地快照 | 数据变化时自动 | 浏览器存储 | 始终加密 | 防误操作 |
| 定时备份 | 到达设定间隔时 | 浏览器存储 + 本地目录 | 始终加密 | 定期保护 |
| 手动导出 | 用户主动操作 | 外部文件 | 可选加密 | 跨设备迁移 |

---

## 自动备份

### 启用自动备份

1. 打开管理页面 → 设置 → 自动备份
2. 开启「启用自动备份」开关
3. 选择备份频率

**备份频率选项：**

| 频率 | 说明 | 适用场景 |
|------|------|---------|
| 每 5 分钟 | 最频繁备份 | 开发测试环境 |
| 每天 | 默认推荐 | 日常使用 |
| 每周 | 较低频率 | 数据变化较少 |
| 每月 | 最低频率 | 静态密钥 |

### 快速路径机制

自动备份采用快速路径设计，**无需解锁**即可正常运行：

**工作原理：**
- 导入/保存密钥时自动生成加密副本
- 备份时直接使用加密副本，无需解密
- 会话过期、浏览器重启后仍可备份
- 支持主密码模式和自定义密码模式

**两种加密模式：**

| 模式 | 说明 | 推荐度 |
|------|------|-------|
| 主密码模式 | 直接复用主密码加密的密钥副本 | ⭐⭐⭐⭐⭐ |
| 自定义密码模式 | 使用独立备份密码加密的副本 | ⭐⭐⭐⭐ |

> 💡 **提示**：无论使用哪种模式，自动备份都无需在触发时输入密码。

### 本地快照

在浏览器存储中保留最近 5 个备份版本：

- **触发时机**：添加/修改/删除密钥后 3 秒，或定时备份
- **保留数量**：最近 5 个版本
- **存储位置**：浏览器本地存储
- **恢复方式**：管理页面 → 备份恢复 → 选择快照

> ⚠️ **注意**：清除浏览器数据会丢失本地快照，请配合目录备份使用。

### 目录备份

自动将备份文件写入本地文件夹：

1. 开启「自动保存到本地目录」
2. 点击「选择目录」授权文件夹
3. 确保权限状态显示「已授权」

**权限状态说明：**

| 状态 | 说明 | 操作 |
|------|------|------|
| 已授权 | 可自动写入 | 无需操作 |
| 需要授权 | 需要用户确认 | 点击「授权」 |
| 权限被拒绝 | 无法写入 | 重新选择目录 |

**文件命名格式：**

```
openpass-backup-2024-01-15T10-30-00.json
```

## 备份加密

### 加密配置

备份加密**始终启用**，可选择两种密码模式：

1. **使用主密码作为备份密码**（推荐）
   - 无需记忆额外密码
   - 直接复用主密码加密副本
   - 设置简单，出错概率低

2. **使用自定义备份密码**
   - 设置独立的备份密码
   - 独立于主密码管理
   - 适合多设备共享备份场景

### 加密算法

| 项目 | 算法 | 说明 |
|------|------|------|
| 密钥派生 | PBKDF2 | 100,000 次迭代 |
| 加密算法 | AES-256-GCM | 认证加密 |
| 盐值 | 随机 16 字节 | 每次加密不同 |

### 加密备份结构

```json
{
  "format": "openpass-backup",
  "formatVersion": 1,
  "appVersion": "0.0.1",
  "exportTime": "2024-01-15T10:30:00.000Z",
  "count": 10,
  "encrypted": true,
  "encryptedData": "Base64编码的加密数据..."
}
```

**说明**：元数据（版本、时间、数量）明文存储，密钥数据加密存储。

## 手动导出

### 操作步骤

1. 打开管理页面 → 备份恢复
2. 点击「立即备份」按钮
3. 文件自动下载到选定目录

> 💡 **提示**：手动备份前需要已解锁，确保能够读取密钥数据。

### 导出位置

- 默认下载目录或自定义备份目录
- 文件名格式：`openpass-backup-日期时间.json`

---

## 导入恢复

### 操作步骤

1. 打开管理页面 → 备份恢复
2. 选择备份文件或本地快照
3. 如果备份已加密，输入解密密码
4. 选择导入方式

### 导入方式

| 方式 | 说明 |
|------|------|
| **合并（跳过重复）** | 保留现有密钥，只添加新密钥 |
| **合并（覆盖重复）** | 用备份中的密钥覆盖现有重复密钥 |
| **合并（两者保留）** | 保留两者，重复的添加后缀 |
| **覆盖** | 删除所有现有密钥，使用备份数据 |

### 版本兼容性

支持导入以下版本的备份：

- 相同主版本号
- 相同或更低的次版本号

例如：v1.2.0 可导入 v1.1.x 和 v1.2.x 的备份。

## 备份最佳实践

### 推荐配置

```
✅ 启用本地快照
✅ 启用目录备份（选择一个专门的备份文件夹）
✅ 使用主密码作为备份密码
✅ 每天备份频率
✅ 定期检查备份状态
```

### 备份检查清单

- [ ] 本地快照已启用
- [ ] 目录备份已授权
- [ ] 备份密码已配置
- [ ] 上次备份时间在预期范围内
- [ ] 备份文件可正常导入

### 灾难恢复

如果丢失所有数据：

1. 重新安装 OpenPass
2. 设置主密码
3. 点击「导入备份」
4. 选择之前导出的备份文件
5. 输入解密密码
6. 选择「覆盖」导入

## 常见问题

**Q: 会话过期后自动备份还能正常工作吗？**

A: 可以。自动备份使用快速路径机制，直接复用已加密的密钥副本，无需解密，会话过期或浏览器重启后仍可正常备份。

**Q: 本地快照和目录备份有什么区别？**

A: 本地快照存储在浏览器中，方便快速恢复，但清除浏览器数据会丢失。目录备份存储在本地文件系统，即使重装浏览器也能保留。

**Q: 为什么目录备份需要授权？**

A: 浏览器安全策略要求，扩展写入本地文件需要用户明确授权。授权一次后，后续自动写入无需再次确认。

**Q: 备份文件可以上传到云盘吗？**

A: 可以。备份文件使用 AES-256-GCM 加密，可安全上传到云存储。但建议使用强密码，并妥善保管密码。

**Q: 如何验证备份是否有效？**

A: 可以在测试环境中导入备份文件验证，或检查备份文件的 JSON 结构是否完整。

**Q: 备份密码忘记了怎么办？**

A: 很遗憾，备份密码无法恢复。建议使用主密码作为备份密码，这样只需记住一个密码。
