# 备份与恢复

OpenPass 提供多层次的备份机制，确保您的密钥数据安全。

---

## 备份策略概览

| 备份类型 | 触发时机 | 存储位置 | 加密 | 用途 |
|---------|---------|---------|------|------|
| 本地快照 | 数据变化时自动 | 浏览器存储 | 可选 | 防误操作 |
| 间隔备份 | 距上次备份超过设定间隔时 | 浏览器存储 + 本地目录 | 可选 | 定期保护 |
| 手动导出 | 用户主动操作 | 外部文件 | 可选 | 跨设备迁移 |

---

## 自动备份

### 启用自动备份

1. 打开管理页面 → 设置 → 自动备份
2. 开启「启用自动备份」开关
3. 选择备份频率（每天/每周/每月）

### 本地快照

在浏览器存储中保留最近 5 个备份版本：

- **触发时机**：添加/修改/删除密钥后 3 秒
- **保留数量**：最近 5 个版本
- **存储位置**：浏览器本地存储
- **恢复方式**：管理页面 → 导入备份 → 选择快照

> ⚠️ **注意**：清除浏览器数据会丢失本地快照，请配合目录备份使用。

### 目录备份

自动将备份文件写入本地文件夹：

1. 开启「自动保存到本地目录」
2. 点击「选择目录」授权文件夹
3. 确保权限状态显示「已授权」

**权限状态说明：**

| 状态 | 说明 | 操作 |
|------|------|------|
| 已授权 | 可自动写入 | 无需操作 |
| 需要授权 | 需要用户确认 | 点击「授权」 |
| 权限被拒绝 | 无法写入 | 重新选择目录 |

**文件命名格式：**

```
openpass-backup-2024-01-15T10-30-00.json
```

## 备份加密

### 启用加密

1. 打开管理页面 → 设置 → 备份加密
2. 开启「启用备份加密」
3. 选择密码类型：
   - **使用主密码**（推荐）- 无需记忆额外密码
   - **自定义密码** - 设置独立的备份密码

### 加密算法

| 项目 | 算法 | 说明 |
|------|------|------|
| 密钥派生 | PBKDF2 | 100,000 次迭代 |
| 加密算法 | AES-256-GCM | 认证加密 |
| 盐值 | 随机 16 字节 | 每次加密不同 |

### 加密备份结构

```json
{
  "format": "openpass-backup",
  "formatVersion": 1,
  "appVersion": "0.0.1",
  "exportTime": "2024-01-15T10:30:00.000Z",
  "count": 10,
  "encrypted": true,
  "encryptedData": "Base64编码的加密数据..."
}
```

**说明**：元数据（版本、时间、数量）明文存储，密钥数据加密存储。

## 手动导出

### 操作步骤

1. 点击扩展图标右上角 **⋮** 菜单
2. 选择「导出备份」
3. 文件自动下载

或在管理页面：
1. 点击「导出备份」按钮
2. 选择是否加密
3. 保存文件

### 导出位置

- 浏览器默认下载目录
- 文件名格式：`openpass-backup-日期时间.json`

---

## 导入恢复

### 操作步骤

1. 点击扩展图标右上角 **⋮** 菜单
2. 选择「导入备份」
3. 选择备份文件
4. 如果备份已加密，输入解密密码
5. 选择导入方式

### 导入方式

| 方式 | 说明 |
|------|------|
| **合并（跳过重复）** | 保留现有密钥，只添加新密钥 |
| **合并（覆盖重复）** | 用备份中的密钥覆盖现有重复密钥 |
| **合并（两者保留）** | 保留两者，重复的添加后缀 |
| **覆盖** | 删除所有现有密钥，使用备份数据 |

### 版本兼容性

支持导入以下版本的备份：

- 相同主版本号
- 相同或更低的次版本号

例如：v1.2.0 可导入 v1.1.x 和 v1.2.x 的备份。

## 备份最佳实践

### 推荐配置

```
✅ 启用本地快照
✅ 启用目录备份（选择一个专门的备份文件夹）
✅ 启用备份加密（使用主密码）
✅ 定期检查备份状态
```

### 备份检查清单

- [ ] 本地快照已启用
- [ ] 目录备份已授权
- [ ] 备份加密已启用
- [ ] 上次备份时间在预期范围内
- [ ] 备份文件可正常导入

### 灾难恢复

如果丢失所有数据：

1. 重新安装 OpenPass
2. 点击「导入备份」
3. 选择之前导出的备份文件
4. 输入解密密码（如果加密）
5. 选择「覆盖」导入

## 常见问题

**Q: 本地快照和目录备份有什么区别？**

A: 本地快照存储在浏览器中，方便快速恢复，但清除浏览器数据会丢失。目录备份存储在本地文件系统，即使重装浏览器也能保留。

**Q: 为什么目录备份需要授权？**

A: 浏览器安全策略要求，扩展写入本地文件需要用户明确授权。授权一次后，后续自动写入无需再次确认。

**Q: 备份文件可以上传到云盘吗？**

A: 如果启用了备份加密，可以上传。但建议使用强密码，并妥善保管密码。

**Q: 如何验证备份是否有效？**

A: 可以在测试环境中导入备份文件验证，或检查备份文件的 JSON 结构是否完整。

**Q: 备份密码忘记了怎么办？**

A: 很遗憾，备份密码无法恢复。建议使用主密码作为备份密码，这样只需记住一个密码。
